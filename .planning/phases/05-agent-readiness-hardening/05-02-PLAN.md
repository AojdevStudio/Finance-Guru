     - [ ] New feature (non-breaking change that adds functionality)
     - [ ] Breaking change (fix or feature that would cause existing functionality to change)
     - [ ] Refactor (code change that neither fixes a bug nor adds a feature)
     - [ ] Documentation (changes to docs only)

     ## Test Plan

     _How were these changes tested?_

     - [ ] Existing tests pass (`uv run pytest`)
     - [ ] New tests added for changed code
     - [ ] Linter passes (`uv run ruff check src/ tests/`)
     - [ ] Type checker passes (`uv run mypy src/ --ignore-missing-imports`)

     ## Checklist

     - [ ] Code follows project conventions (3-layer architecture, type hints, docstrings)
     - [ ] No hardcoded personal data or credentials
     - [ ] Changes are covered by tests
     - [ ] Documentation updated (if applicable)
