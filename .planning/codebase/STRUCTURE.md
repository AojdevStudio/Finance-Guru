## Naming Conventions

### Python Files

**Calculator Classes** (Layer 2):
- File: `src/{analysis|strategies|utils}/{tool_name}.py`
- Class: `PascalCase` (e.g., `RiskCalculator`, `MomentumIndicators`)
- Example: `src/analysis/risk_metrics.py` contains `RiskCalculator` class

**CLI Interfaces** (Layer 3):
- File: `src/{analysis|strategies|utils}/{tool_name}_cli.py`
- Pattern: Tool name + `_cli` suffix
- Example: `src/analysis/risk_metrics_cli.py` for RiskCalculator CLI

**Pydantic Models** (Layer 1):
- File: `src/models/{tool_name}_inputs.py`
- Classes: Input + Output models
- Pattern: `{Tool}Input`, `{Tool}Output`, `{Tool}Config`
- Example: `PriceDataInput`, `RiskMetricsOutput`, `RiskCalculationConfig`

### Analysis Reports

**Report Naming** (in `fin-guru-private/fin-guru/analysis/`):
- Analysis reports: `{topic}-{YYYY-MM-DD}.md`
  - Example: `tsla-momentum-2025-02-01.md`
- Buy recommendations: `buy-ticket-{YYYY-MM-DD}-{short-descriptor}.md`
  - Example: `buy-ticket-2025-02-01-hybrid-drip-v2.md` (max 2-3 words)
- Strategy documents: `{strategy-name}-master-strategy.md` (no dates)
  - Example: `hybrid-dividend-drip-master-strategy.md`
- Monthly reviews: `monthly-refresh-{YYYY-MM-DD}.md`
  - Example: `monthly-refresh-2025-02-01.md`

### Agent Definitions

**Agent Files** (in `.claude/commands/fin-guru/agents/`):
- Format: `{agent-name}.md`
- Examples:
  - `finance-orchestrator.md` - Master orchestrator
  - `market-researcher.md` - Market intelligence
  - `quant-analyst.md` - Quantitative analysis

### Task Definitions

**Task Files** (in `fin-guru/tasks/`):
- Format: `{task-name}.md`
- Examples:
  - `research-workflow.md` - Multi-step research
  - `quantitative-analysis.md` - Quant analysis
  - `strategy-integration.md` - Strategy planning

## Data Flow Paths

**CLI Execution**:
```
src/analysis/risk_metrics_cli.py
  → Fetch data: src/utils/market_data.py
  → Validate: src.models.risk_inputs.PriceDataInput
  → Calculate: src/analysis/risk_metrics.RiskCalculator
  → Format: src.models.risk_inputs.RiskMetricsOutput
  → Output: JSON or text
```

**Agent Workflow**:
```
.claude/commands/fin-guru/agents/finance-orchestrator.md
  → Route to specialist: fin-guru/workflows/route-to-agent/workflow.yaml
  → Execute task: fin-guru/tasks/{task_name}.md
  → Call CLI: uv run python src/{module}/{tool}_cli.py
  → Create artifact: fin-guru/templates/{template}.md
  → Save to: fin-guru-private/fin-guru/analysis/{report_name}.md
```

## Key File Locations Reference

### Where to add new financial tools

1. **Step 1**: Create input/output models in `src/models/{tool_name}_inputs.py`
2. **Step 2**: Export in `src/models/__init__.py`
3. **Step 3**: Create calculator in `src/analysis/{tool_name}.py` (or `src/strategies/` or `src/utils/`)
4. **Step 4**: Create CLI in `src/analysis/{tool_name}_cli.py`
5. **Step 5**: Add to root `CLAUDE.md` tool reference table
6. **Step 6**: Update `AGENTS.md` with agent-tool matrix

### Where portfolio configurations live

- Layer assignments: `src/config.py` (default_layers dict)
- Override path: `~/.config/finance-guru/layers.yaml`
- Runtime config: `fin-guru/config.yaml`

### Where analysis outputs go

- Generated reports: `fin-guru-private/fin-guru/analysis/`
- Research archive: `fin-guru-private/research/`

## Environment Setup

**Key Environment Files**:
- `.env` (git-ignored) - Local secrets and API keys
- `.env.example` - Template for required variables
- `.python-version` - Python 3.12 specification
- `pyproject.toml` - Dependency definitions

**Dependencies installed with**:
```bash
uv sync              # Install from lock file
uv add package_name  # Add new package
```

**Running Python code**:
```bash
uv run python src/analysis/risk_metrics_cli.py TSLA --days 90
uv run pytest tests/python/
```

## Project Configuration Hierarchy

1. **Root project level**: `pyproject.toml`, `setup.sh`, `CLAUDE.md`
2. **Python layer**: `src/config.py` (FinGuruConfig class)
3. **Runtime config**: `fin-guru/config.yaml` (portfolio layers, settings)
4. **Local override**: `~/.config/finance-guru/layers.yaml` (user config)
5. **Environment vars**: `.env` file for secrets/API keys

## Git Ignore Patterns

**Ignored directories** (see `.gitignore`):
- `.venv/`, `node_modules/` - Dependencies
- `__pycache__/`, `.pytest_cache/`, `.ruff_cache/` - Build artifacts
- `.uv-cache/` - Package manager cache
- `.DS_Store` - macOS metadata
- `.env` - Secrets (but `.env.example` is tracked)

**Tracked** (important):
- `.gitignore` - Ignore rules
- `.gitattributes` - Git attributes
- All source code in `src/`, `.claude/`, `fin-guru/`, `fin-guru-private/`
